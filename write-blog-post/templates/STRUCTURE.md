# Blog post structure

Use this skeleton unless the user asks for a different format.

## Frontmatter

See `FRONTMATTER.md` for the exact fields.

## Body

```
# {H1: clear, keyword-rich title}

{2–3 sentence intro. Hook the reader, state who this is for, promise the payoff.}

## {H2: What is / Why it matters}
{Frame the problem.}

## {H2: Core section — the main answer or method}
{The meat of the post. Break into H3 subsections if long.}

### {H3: Sub-point}
### {H3: Sub-point}

## {H2: Step-by-step (if applicable)}
{Numbered list with clear actions.}

## {H2: Examples / Case studies}
{Concrete proof. Cite sources.}

## {H2: Common mistakes}
{2–4 pitfalls and how to avoid them.}

## {H2: Conclusion}
{Recap the key takeaway in 2–3 sentences.}

> **Ready to {next step}?** {Call to action linking to product, demo, or related post.}
```

## Length guidance

- Listicle / how-to: 1,200–1,800 words.
- Pillar page: 2,000–3,000 words.
- Thought leadership: 800–1,200 words.

When in doubt, ask the user for the target length.
